In Bento, tweens can be used to interpolate one value to another within a set amount of time.

Tweens are programmed like so:


new Tween({
  from: 0,
  to: 100,
  in: 60,
  ease: Tween.LINEAR,
  onUpdate: function (v, t) {
    // for example, tweening the x position
    entity.position.x = v;
  },
  onComplete: function () {
    // stuff to happen when tween ends
  }
});


Note how the actual usage of the tweened value is applied in onUpdate. This is much more flexible than tweening a set of properties.

Seriously, what's up with all the Javascript game engines implementing tweens as
Tween(object).from(properties, value).to(properties, value).start() anyway???
We don't all need to copy from TweenJS and GreenSock, those are targeted to tweening DOM elements.
